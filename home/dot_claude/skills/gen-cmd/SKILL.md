# gen-cmd - CLIコマンド生成・検証

自然言語の説明からCLIコマンドを生成し、実際に動作確認してから返す。

## コンテキスト

- OS: !`uname -s -r`
- シェル: !`echo $SHELL`
- アーキテクチャ: !`uname -m`

## ワークフロー

### Step 1: コマンド生成

ユーザーの自然言語入力を解析し、適切なCLIコマンドを生成する。

**注意事項:**
- macOS (BSD) 互換のコマンドを使用する（GNU拡張に依存しない）
  - 例: `find` のBSD版では `-printf` は使えない、`sed -i` には `''` が必要
- パイプや複合コマンドも可
- 必要に応じて `grep`, `find`, `awk`, `sed`, `jq`, `curl` 等を組み合わせる

### Step 2: 安全性チェック

以下の破壊的コマンドが含まれる場合は**実行せず**、コマンドだけを出力して警告する:
- ファイル操作: `rm`, `mv`, `cp`（上書き）, `chmod`, `chown`
- ディスク操作: `dd`, `mkfs`, `fdisk`
- システム操作: `kill`, `killall`, `shutdown`, `reboot`
- データ操作: `truncate`, `shred`
- リダイレクトによる上書き: `> file`（`>>` は許可）

破壊的コマンドと判定した場合は、検証をスキップしてStep 4へ進む。

### Step 3: 動作確認

1. 生成したコマンドをBashツールで実行する（タイムアウト: 3000ms）
2. 結果を評価する:
   - **成功**: exit code 0 かつ期待通りの出力 → Step 4へ
   - **失敗**: エラー出力を解析し、コマンドを修正して再実行（最大3回）
3. 3回リトライしても失敗した場合:
   - 最も妥当なコマンドを選択してStep 4へ
   - コマンドの後に `# 注意: 動作確認で失敗` とコメントを付ける

### Step 4: 出力

**検証成功時:**
コマンド文字列のみをプレーンテキストで出力する。マークダウンのコードブロックや説明は一切付けない。

```
出力例:
find ~/code/xxx -name "foo" -type f
```

**破壊的コマンドの場合:**
コマンド文字列の後に改行して警告を1行付ける。

```
出力例:
rm -rf ~/tmp/cache
# ⚠ 破壊的コマンドのため動作確認をスキップしました
```

**3回失敗した場合:**
最善のコマンドとエラー概要を付ける。

```
出力例:
some-command --flag
# ⚠ 動作確認失敗: command not found
```

## 制約

- 出力は必ずコマンド文字列で始めること（説明文を先に出さない）
- マークダウンのコードブロック（```）で囲まない
- 複数行コマンドの場合は `\` で継続するか `;` / `&&` で連結する
- コマンドの説明や解説は付けない（コメント行での警告のみ例外）
